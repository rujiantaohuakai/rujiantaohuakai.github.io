条件变量和读写锁

一、条件变量 condition_variable

1.1 什么是条件变量

条件变量是一种同步原语，用于在线程之间协调共享资源的访问。它允许一个线程等待特定条件的满足（如某个值的变化），而另一个线程在条件满足时通知（或唤醒）等待的线程。这种机制可以防止线程忙等待，从而提高系统效率。

特点：

  1. 等待和通知机制：线程可以等待某个条件的改变，而不需要一直占用CPU资源。
  2. 与互斥锁配合使用：通常与互斥锁一起使用，以保护共享数据的访问。
  3. 线程通信：实现线程之间的高效通信和协调

1.2 条件变量的作用

  1. 减少轮询从而提高效率：没有条件变量，CPU会浪费时间反复轮询某一个条件。条件变量出现使得线程可以在不满足条件进行休眠，将资源让给有需要的其他线程；
  2. 线程之间的协调：两个线程之间执行可以变得有序，也就是一个线程执行完成后，另外一个线程才会执行；
  3. 复杂同步条件：有时候一个线程不仅需要知道资源可达，还需要知道一些额外信息，比如说生产消费者中，缓冲队列是否满之类的操作；

线程进入阻塞可以是sleep，也可以是wait。sleep期间所持有的锁不会释放，到时见会自动唤醒，而wait期间将会释放锁，唤醒还需要额外的条件。前者更多的用在模拟操作延迟，后者用于多线程协作场景。

1.3 如何定义条件变量

在C++中，可以使用std::condition_variable来定义和使用条件变量。条件变量通常与互斥锁（如std::mutex）一起使用，以确保在检查和修改共享数据时线程间的同步。

示例代码：

// 条件变量condition_variable
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
#include <vector>
#include <chrono>

std::mutex mtx; // 互斥锁
std::condition_variable cv; // 条件变量
bool flag = false; // 共享的变量 用于检测条件是不是成立

// 根据共享变量组建逻辑
void print_num(int num){
    std::unique_lock<std::mutex> ulock(mtx); // 加锁
    // while(!flag){ // 等待条件成立
    //     cv.wait(ulock); // 等待条件成立，进入阻塞状态，释放互斥锁，等待唤醒
    // }
    cv.wait(ulock, []{return flag;}); // 第二个参数接受一个谓词，返回false时阻塞。这样也可以，和上面的while循环效果一样
    //std::this_thread::sleep_for(std::chrono::seconds(1)); // 模拟耗时操作
    std::cout << "thread id: " << std::this_thread::get_id() << "-" << "num " << num << std::endl; // 条件成立，打印  
}

// 更新共享变量
void updateflag(){
    std::cout << "this is updateflag()" << std::endl;
    std::this_thread::sleep_for(std::chrono::seconds(3));
    std::unique_lock<std::mutex> ulock(mtx);
    flag = true; // 条件成立
    cv.notify_all(); // 唤醒所有等待线程
}

std::condition_variable cv2; // 条件变量
bool flag2 = false; // 共享的变量 用于检测条件是不是成立

void wait_for_test(){
    std::unique_lock<std::mutex> lck(mtx);
    if(cv2.wait_for(lck, std::chrono::seconds(3)) == std::cv_status::no_timeout){ // 等待三秒，第一个线程不会超时，第二个会超时
        std::cout << "Thread " << std::this_thread::get_id() << " condiction is true and no timeout" << std::endl;
        std::this_thread::sleep_for(std::chrono::seconds(1)); // 模拟耗时操作，持有锁1s
    }
    else{
        std::cout << "Thread " << std::this_thread::get_id() << " timeout" << std::endl;
    }
}

void updateflag2(){
    std::cout << "this is updateflag2()" << std::endl;
    std::this_thread::sleep_for(std::chrono::seconds(2));
    std::unique_lock<std::mutex> lck(mtx);
    flag2 = true; // 条件成立
    cv2.notify_one(); // 唤醒一个等待线程
    cv2.notify_one(); // 唤醒另一个等待线程
}

int main(){

    std::vector<std::thread> threads;
    for(size_t i = 0; i < 10; i++){
        threads.emplace_back(print_num, i);
    }

    updateflag(); // 更新共享变量，使条件成立

    for(std::thread &t : threads){
        t.join();
    }

    // wait_for_test(); // 测试wait_for
    std::thread t1(wait_for_test);
    std::thread t2(wait_for_test);
    updateflag2(); // 更新共享变量，使条件成立

    t1.join();
    t2.join();

    return 0;
}

在这个示例中：

  1. cv.wait():
  • 定义一个 std::mutex 对象 mtx 来保护共享变量 flag。
  • 定义一个 std::condition_variable 对象 cv 用于线程间的等待和通知。
  • 调用 print_num 函数的线程会检查 flag 变量，如果它不为 true，则调用 cv.wait(ulock) 来等待。在等待期间，线程会释放锁（由 std::unique_lock 管理），并阻塞直到其他线程通过调用 cv.notify_all() 或 cv.notify_one() 来唤醒它。
  • updateflag 函数模拟了一个操作，该操作在一段时间后设置 flag 为 true，并通知所有等待的线程。
  • 在 main 函数中，创建了 10 个线程，每个线程都调用 print_num 函数。然后，调用 updateflag 函数来触发条件变量，最后等待所有线程完成。
  1. cv.wait_for():
  • 使用前面定义的 std::mutex 对象 mtx 来保护共享变量 flag2。
  • 定义一个 std::condition_variable 对象 cv2 用于线程间的等待和通知。
  • 调用 wait_for_test 函数的线程
  • updateflag2 函数模拟了一个操作，该操作在 2 秒后设置 flag2 为 true，并通知两个等待中的线程。
  • 在 main 函数中，创建了 2 个线程，每个线程都调用 wait_for_test 函数。然后，调用 updateflag2 函数来触发条件变量，最后等待所有线程完成。

1.4 wait和wait_for

在 C++ 的多线程编程中，std::condition_variable 类提供了 wait 和 wait_for 方法，用于让线程在满足特定条件之前等待。这两个方法都与互斥锁（std::mutex 或 std::unique_lock）一起使用，以确保线程安全地访问共享数据。

1.4.1 wait方法

  1. wait可以有两个参数：wait(lock); wait(lock, predicate);
  • std::unique_lock<std::mutex>& lock：一个与互斥锁相关联的 std::unique_lock 对象。调用 wait 时，这个锁会被自动释放，允许其他线程获取该锁并执行。当 wait 返回时，锁会被重新获取。
  • Predicate pred：一个可调用对象（如函数、lambda 表达式等），它返回一个布尔值，false 时继续等待，true 时退出 wait，继续执行后续代码。
  1. wait(lock) 工作原理：一般与 while(!flag) 循环结合使用，检查flag->flag为true->继续执行；flag为flase->进入while执行wait(lock)->释放锁->阻塞线程。
  • 调用 wait(lock)时，直接释放互斥锁，然后线程进入阻塞状态，等待条件变量被通知。
  1. wait(lock, predicate) 工作原理：检查条件->条件成立->尝试获取锁->获取成功->再次检查条件->条件成立->继续执行；条件不成立->释放锁->阻塞线程。
  • wait(lock, predicate)会先检查 predicate，如果 predicate 为 true，则继续执行，如果 predicate 为 false，则释放锁并阻塞，等到被唤醒时先尝试获取锁，如果获取失败，则继续阻塞，等到锁被释放，再自动尝试获取锁，

注意：

  • 线程执行到 cv.wait() 时，一定（必须）已经持有锁 lock，否则会抛出异常。
  • 如果是在被唤醒后尝试获取锁失败，则继续阻塞，等待锁被释放后，不用通知，会自动尝试获取锁，直到获取成功。
